Vim 使わせていただいております。

私はエンジニアになって1年3ヶ月ほどしかたっておりません。

そのため、すべてが模索中です。
エディタも模索中です。

自分自身の開発効率をより良いものにするために良いものはないかと
広大なネット空間を飛び回っております。

そんなこんなで現在は Vim をベースとした エディタ NeoVim を使用しています。

Vim と NeoVim の違いの話はネット上にいくらでも転がっているので、 どペーペーエンジニアに影響を与え、100% Vimで開発を行うようにさせてしまった記事を紹介します。

Vim でこの上なく参考になった記事の面々

現在、私は主にNeoVimを使っています。
かなり良いです。

エンジニアになりたてのときはAtomを使っていましたが
その時からVimのキーバインドに変更するプラグインを使用していました。

ですが、(今でも全然使えていませんが)Vim の機能の3%も使えていませんでした。

そのためいろいろな記事を読みました。
その中でも、私に青天の霹靂とも言える衝撃を与えたオススメの記事を紹介します。

1. さっさと帰りたい怠け者エンジニアは vim をマスターしましょう その1 - 基本と移動

Vim のキーバインドをどのように組み合わせて実際に開発を進めていけばよいのかというユースケースがハンズオンで学べます。

この記事で書かれていることを手になじませれば基本的にコードを書くということに困ることはないと思います。

キーバインドチートシート等をいきなり見るよりはるかに良いと思います。 その2も合わせて読むべきです。

2. Vim-Galore : Vimについて知っておくべき全てのこと1~5

かなりの文章量です。

そのため、 Vim でこういうことやりたいなと思ったときに
知識のインデックスが貼られているのですぐに情報にたどり着けるようになります。

操作、設定どちらにておいても有用です。

3. Vim のすゝめ 第3回 Vim を使いこなすために必要なたった一つの能力

骨子は

:help で調べる癖を付けましょう。

Vimについて知っておくべき全てのこと を読んでおくことで

:help hogehoge の勘所が身につきます。

優秀なエンジニアは一次ソースに当たるという話ですね。 私も常にそうでありたい。
